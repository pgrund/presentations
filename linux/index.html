Processes
Linux is a multi tasking operating system: multiple processes - those are programs, after loaded to memory and started - run in parallel.

This actually means, that the operating system splits available computation time of the processor in small time slots (milliseconds), which are provided to severale processes. This task is accomplished by a central instance - scheduler. He manages which process gets which timeslot.


Therefore, no single process gains total control of all computation time and any "freezed" process cannot shutdown the whole system.

Users
Linux is also a multi-user operating system, multiple users can operate on the same machine on several terminals at the same time.

A user is identified by its username.
Internally, all users are managed via their unique userID (UID).

Each user is at least member of one group. There might be many different groups in the system - internally handled via groupID (GID).


Generally speaking, groups are just an opportunity to restrict access in more detailed way (see file access on the next slides ...).

Root

User with userID 0 has a special role - he's root.
Root has no security constraint by the system - in short - he is allowed to do anything.

Root is capable of destroying the whole system with only one command. He can delete all the work of a week, a month ... That's the reason, why the system administrator logs on with his regular user - he becomes root only for administrative tasks, which needs special access rights.

Substring removal
${PARAMETER#PATTERN}, ${PARAMETER##PATTERN}
removes specified pattern (see matching) from the beginning
${PARAMETER%PATTERN}, ${PARAMETER%%PATTERN}
removes specified pattern (see matching) from the end

single form (#, %) removes shortest text matching, double form (##, %%) removes longest matching

$ MYSTRING="Be liberal in what you accept, and conservative in what you send"
$ echo ${MYSTRING#* }
liberal in what you accept, and conservative in what you send
$ echo ${MYSTRING##* }
send
Substring offset
${PARAMETER:OFFSET}, ${PARAMETER:OFFSET:LENGTH}
expand only part of parameter's value, given a position to start and an optional length
If LENGTH is omitted, the parameter will be expanded up to the end of the string
If LENGTH is negative, it's taken as a second offset into the string, counting from the end of the string
OFFSET and LENGTH can be arithmetic expression
$ echo ${MYSTRING:34}
conservative in what you send
$ echo ${MYSTRING:34:13}
conservative
$ echo ${MYSTRING: -13:5}
what
$ echo ${MYSTRING:(7/2):7}
liberal
Warning!

The OFFSET starts at index 0, not at 1!
When using a negative offset you need to separate the (negative) number from the colon: ${MYSTRING: -10:5}, ${MYSTRING:(-10):5}.

Search and replace
${PARAMETER/PATTERN/STRING}, ${PARAMETER//PATTERN/STRING}
replace a substring matched by pattern
single-form (/) replace first occurrence, double-form (//) replace all occurrences.
if STRING is ommitted, matched pattern will be removed
$ echo ${MYSTRING/in/by}
Be liberal by what you accept, and conservative in what you send
$ echo ${MYSTRING//you}
Be liberal in what accept, and conservative in what send

Command Interpretation
As stated at the beginning, a command gets not executed directly by the shell, it interprets the command first according to the following order ...
  1. seperate command line to words
  2. substitute command line
  3. redirection of input and output streams
  4. search for command (type and place)
  5. execute the command

Substituion can change a commandline tremendously.
It is important to understand how this change happens (error messages, side effects, ...).

Substitution (Part I)
1 bracket substition
e.g. create a systematic series of filenames
$ echo /home/{maier,{b,c,f}huber}
  /home/maier  /home/bhuber  /home/chuber  /home/fhuber
2 tilde substitution
tilde (~) gets replaced by home directory
$ cp foo ~
$ cp foo ~bhuber
Substitution (Part II)
3 variable substitution
replace variables/parameter by its value
$ echo ${JAVA_HOME//\//.}
4 command substitution
using output of a command as part of another command line (insert)
$ echo Following users have a home directory: `ls /home`
Following users have a home directory: grund foo jack
bash provides an another syntax, capable of nesting as well
$ echo Following users have a home directory: $(ls /home)
Substitution (Part III)
5 arithmetic substitution
bash allows an integer arithmetic
$ echo one plus one: $[1+1]
one plus one: 2
6 seperate words (again)
only if expression is not within single ('...') or double ("...") quotes
7 wildcard substitution
according to wildcards as stated above
$ echo /home/*
/home/maier  /home/bhuber ....
Substitution (Part IV)
8 demasking
drop all masking characters - if not masked on their own (/, ', ")

summary
1 bracket {} 2 tilde ~ 3 variable $
4 command `..` or $(...) 5 arithmetic $[]
6 words '',"" 7 wildcard * 8 demasking /,',"

Input / Output Streams

For each command execution there always 3 channels:

stdin command stdout
stderr
stdin
standard input assigned per default to the keyboard
channel number 0
stdout
standard output assigned per default to the display
channel number 1
stderr
standard error assigned per default to the display as well
channel number 2

Channel numbers 0, 1 and 2 are actually special file descriptors (allow programs to access opened files) since they are available all the time.

Pipe

The pipe symbol (|) connects commands:

  • stdout of first command gets connected to stdin of second command.
  • only works, if first command writes to channel 1 and second command reads from channel 0

Jobs running in background mode
  • Imagine you just started some long running job in your shell and after some 20s you get soooo bored watching it e.g. 
    $ find / -name '*e*' -type f
  • What can you do? Just get on with your work while it runs in the background!
  • Just type [Ctrl]+[z] that should return [1]+ Angehalten find ... then type 
    $ bg %1
    where %1 is the id of the job, that the [Ctrl]+[z] returned
Background jobs
  • But(!) now your job still writes to your terminal!!
  • better to write it then like that 
    $ find / -name '*e*' -type f >/tmp/files-w-e.log 2>/dev/null
  • then [Ctrl]+[z]
    $ ...
[2]+  Angehalten find / -name...
$ bg %2
  • and yet better: send it to the background when you start it by appending & to the command 
    $ grep -Ri 'error' /var/log >/tmp/errors.txt 2>/dev/null &

Bash Scripting
Definition

A shell program, called a script, is an easy-to-use tool for building applications by "gluing together" system calls, tools, utilities, and compiled binaries. Virtually the entire repertoire of UNIX commands, utilities, and tools is available for invocation by a shell script.

Advanced Bash-Scripting Guide

Additionally, bash provides for usage within scripts:

  • internal shell commands
  • tests
  • loops
When NOT TO USE shell scripts:
  • Resource-intensive tasks, especially where speed is a factor (sorting, hashing, recursion ...)
  • Procedures involving heavy-duty math operations, especially floating point arithmetic, arbitrary precision calculations, or complex numbers
  • Cross-platform portability required
  • Complex applications, where structured programming is a necessity (type-checking of variables, function prototypes, etc.)
  • Mission-critical applications upon which you are betting the future of the company
  • Situations where security is important, where you need to guarantee the integrity of your system and protect against intrusion, cracking, and vandalism
  • Project consists of subcomponents with interlocking dependencies
  • Extensive file operations required (Bash is limited to serial file access, and that only in a particularly clumsy and inefficient line-by-line fashion.)
  • Need native support for multi-dimensional arrays
  • Need data structures, such as linked lists or trees
  • Need to generate / manipulate graphics or GUIs
  • Need direct access to system hardware or external peripherals
  • Need port or socket I/O
  • Need to use libraries or interface with legacy code
  • Proprietary, closed-source applications (Shell scripts put the source code right out in the open for all the world to see.)
